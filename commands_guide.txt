# 网站维护常用命令指南

本文档包含了从启动本地预览到推送到GitHub的完整命令流程。

### 1. 设置正确的Ruby环境 (每次打开新终端时，建议先执行一次)

确保终端使用的是您通过Homebrew安装的Ruby版本，而不是系统自带的旧版本。

```bash
export PATH="/opt/homebrew/opt/ruby/bin:$PATH"
```

### 2. 安装/更新依赖 (仅在初次设置或遇到依赖问题时需要)

如果提示缺少gem包，请运行：
```bash
bundle install
```

### 3. 本地预览网站

在修改网站内容后，您可以使用此命令在本地电脑上实时预览效果。

# 清理缓存 (如果预览没有实时更新，请先运行此命令)
```bash
bundle exec jekyll clean
```

# 启动服务器
```bash
bundle exec jekyll serve
```
启动后，在浏览器中打开 `http://localhost:4000` 即可查看。
按 `Ctrl + C` 停止服务器。

### 4. 提交并推送到GitHub

当您对本地预览的效果满意后，使用以下命令将所有更改发布到您的线上网站。

# 第一步: 查看文件状态 (可选，但推荐)
```bash
git status
```

# 第二步: 添加所有更改到暂存区
```bash
git add .
```

# 第三步: 提交更改并添加描述信息
```bash
git commit -m "这里写本次修改的摘要，例如：更新项目介绍"
```

# 第四步: 推送到GitHub
```bash
git push
```
推送成功后，等待几分钟，您线上的GitHub Pages网站就会更新。
